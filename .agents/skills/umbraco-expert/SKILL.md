# Umbraco Expert

## Overview
This skill provides procedural knowledge for developing and maintaining the Umbraco-based backend of the JuulTimesedler application. It focuses on the Umbraco 13+ (net9.0) ecosystem, ensuring idiomatic implementation of CMS logic and API integrations.

## Core Tasks

### 1. Content Modeling & Management
When designing or updating Document Types, Member Types, or Media Types:
- **ModelsBuilder:** Use strongly-typed models generated by Umbraco.
- **Content Service:** Use `IContentService` for programmatic content creation and updates.
- **Composition:** Prefer compositions for shared properties (e.g., SEO, Audit).

### 2. API Development (Umbraco Controllers)
Implement controllers following Umbraco patterns:
- **SurfaceControllers:** For handling front-end form submissions (e.g., `TimesheetController`).
- **UmbracoApiController:** For exposing RESTful APIs to the Blazor front-end (e.g., `ProjectController`).
- **Dependency Injection:** Inject `IUmbracoContextAccessor`, `IPublishedContentQuery`, and custom services (e.g., `ITimesheetService`).

### 3. Service Layer Integration
Consolidate business logic into specialized services in `JuulTimesedler_BE/Services/`:
- **TimeService:** Logic for calculating durations and verifying workday rules.
- **WorkersService:** Integration with Umbraco Member service for user management.
- **TimesheetService:** CRUD operations for timesheet entries.

### 4. Configuration & Settings
Manage environment-specific settings in `appsettings.json`:
- **Umbraco Cms:** Configure database connections, ModelsBuilder modes, and security settings.
- **Custom Settings:** Use the Options pattern (`IOptions<T>`) for project-specific configurations.

## Best Practices
- **Naming Conventions:** Use PascalCase for Document Type aliases and property aliases.
- **Security:** Always validate user permissions before performing content operations.
- **Efficiency:** Use `IPublishedContent` (the cache) for read operations; use services for write operations.
